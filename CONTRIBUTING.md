# Contributing / 参与

Thanks for your interest! 🙌 / 感谢关注！

**This repository hosts the built static app** (the `dist/` output) — so you can self-host it and so issues/stars have a home. The development source lives in a separate repo and is synced here on each release.

**本仓库存放的是打包后的静态产物**（`dist/`），方便你自托管、也给 issue / star 一个落脚点。开发源码在另一个仓库，每次发布时同步过来。

## Best ways to help / 最有用的参与方式

- 🐛 **Report bugs / 反馈问题** — open an [Issue](../../issues/new/choose). For a BYOK tool the #1 detail is **which endpoint / proxy** you used, so please include it. **Never paste your API key.** / BYOK 工具里最关键的信息是**你用的哪个端点 / 中转**，请务必写上。**不要粘贴 API Key。**
- 💡 **Request features / 提需求** — open an [Issue](../../issues/new/choose) with *what* + *why*.
- ⭐ **Star** the repo if it's useful — it genuinely helps. / 觉得有用点个 star，真的有帮助。
- 🔁 **Share it / 帮忙分享** — especially the voice flow; a short clip is worth a thousand words. / 尤其是语音生成，一段短视频胜过千言。

## Pull requests

Because this repo contains **generated build output**, code PRs against `dist/` can't be merged — they'd be overwritten on the next sync. If you'd like to contribute code, open an issue first and we'll find the best path.

本仓库是**构建产物**，针对 `dist/` 的代码 PR 无法合并（下次同步会被覆盖）。想贡献代码请先开 issue 沟通。

## Privacy / 隐私

This app is 100% client-side and BYOK — your key, prompts, and images never touch a server. Please keep that in mind when filing issues: don't share keys, and feel free to redact prompts.

应用纯客户端、BYOK——密钥 / prompt / 图像都不经过任何服务器。提 issue 时别贴 key，prompt 可自行打码。
